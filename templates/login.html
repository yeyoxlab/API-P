About application-only auth

Tokens are passwords

Keep in mind that the consumer key & secret, bearer token credentials, and the bearer token itself grant access to make requests on behalf of an application. These values should be considered as sensitive as passwords and must not be shared or distributed to untrusted parties.

SSL absolutely required

This manner of authentication is only secure if SSL is used. Therefore, all requests (both to obtain and use the tokens) must use HTTPS endpoints, which is also a requirement of using API v1. Please follow the best practices detailed in Connecting to Protcloud API using SSL - peers should always be verified.

Issuing application-only requests

First of all, it necessary to encode the secret key with SHA-128 then use it as credentials to obtain a bearer token.


FIELD VALUE
NAME_APP My_app
SECRET_KEY xvz1evFS4wEEPTGEFPHBog
SECRET_KEY WITH SHA-1 25e72cd75d2d3e0c1061819682c97abce19a3e84

Obtain a bearer token

Once you calculated the SHA-1 of the secret key, you will use it in a new request.

  • The request must be a HTTP POST request.
  • The request must be include name_app and secret key
